C++模拟Penney's game（欺负人的小游戏

发表于 2019-12-01

Penney’s game

Penney’s game是一个两个玩家之间生成用“正面”和“反面”为代称产生二进制数列的游戏。——Wikipedia

存在着不太符合直觉的地方，蛮有趣的

解释与简单证明

举个例子，A和B两个人不停的扔硬币玩。如果序列中先出现“正正反”算A赢，先出现“反正正”算B赢。凭着直觉来看的话，是不是还算公平？毕竟扔三次硬币，两者出现的概率都是1/8，而且两者又是互相取反的关系，似乎没什么问题。

それは違うぞ！这个游戏是不公平的，B获胜的概率要远大于A，是A的3倍。为什么会这样子呢？虽然“正正反”与“反正正”在一串序列中（序列足够长）出现的概率是相等的。但在这个游戏中，存在一个先后的关系，先出现序列的人就赢了。

证明一下：前两次扔硬币可能的结果分别是：“正正”、“正反”、“反反”、“反正”。那么只有结果为“正正”的时候，A才（有机会）获胜，因为剩下的三种情形里A想要获胜的话之后必须出现“正正”或者“正”，而因为前面有一个“反”的存在，B永远比A先达成条件获胜。

所以说在这种情况下，前两次的结果就已经决定了游戏的结局。结论就是B获胜的概率是A的三倍。

别的情况就不解释了，都是一个道理，本质就是出现的先后顺序，当A猜测的前两字节刚好对上时，B的后两字节也对上。

小知识

Penney的游戏是一个非传递博弈，什么是非传递博弈呢？
举个例子：石头剪刀布。A > B, B > C 不能得出A > C的结论。
Penney’s game 也是一样的，不能通过制胜关系来寻找策略。

那么它的最优解应该怎么找呢？可以参考这个表格

2号玩家的策略：第一个字节与1号玩家的第二个字节相反，第二个字节与1号玩家的第一个字节相同，第三个字节与1号玩家的第二个字节相同。
